# 2026-07-12 · Forge icons, resources & detail drawer — design

**Status: design approved in session (M, 2026-07-12, post-bite-2 review). Read top to bottom before the build session starts. Builds on the shipped bite-2 bar (`forge-kit-derive.js` / `forge-feed-render.js` / `forge-hud.js`, flag `USE_FORGE_BAR`).**

This is the presentation-layer arc the bite-2 review surfaced: real icons everywhere with player choice, full descriptions without a mid-turn sheet dive, and resources as a first-class HUD surface. It also lands M's Action Surge ruling and the sheet-order verification.

## 0 · What ships (one sentence each)

1. **`spell-icons.js`** — a sibling module to `item-icons.js` (~120–150 curated game-icons.net glyphs, same contract: `BODIES`/`CATEGORIES`/`iconFor`/`iconSvg`, 512 viewBox, `currentColor`, CC-BY 3.0 attribution) covering spells, class features, and maneuvers.
2. **The two-registry resolver** in forge-kit-derive.js — `resolveIcon(tile)`: inline `icon` → `structural.iconOverrides[tileId]` → keyword seed → category default → letter tile; SpellIcons checked before ItemIcons; weapon tiles inherit the inventory item's icon (the bag and the bar always match).
3. **The hand-drawn sprite block deleted** from the mock; all tiles render via `iconSvg()` from the real registries.
4. **Bonus tab = bonus + free** (M's ruling: "everything that doesn't eat your action") — Action Surge joins the tab; one filter change in `bonusTiles`.
5. **Resource passthrough + the ⬡ Resources tab** — full params (`recharge`, `die`, `tag`, `origin`, `source`, `custom`) ride each pool; top row curated to economy + move + ≤3 pip groups + `+N` overflow chip; the tab shows scrollable cards with tap-to-expand detail.
6. **The detail drawer** — long-press / right-click any tile: name, cost line, full description (spell text lazy-loaded), Change icon, Hide from bar.
7. **Shared `icon-picker.js`** — gear-manager's picker pattern extracted into a widget, hosted in the forge drawer and the sheet action editor, writing to the one storage contract.
8. **Sheet-order verification** — a smoke proving `assembleActions` order survives into `tabs.attacks`; fix in derive if it doesn't.

Out of scope, named so nobody re-derives it: Supabase feed insert (still the top carry-forward from the bite-2 spec, its own bite); universal-action fact publishing (protocol review needed); appearance prefs (§6.5 of the bite-2 spec); drag-reorder in the forge (sheet order is authoritative; revisit later); manual resource adjust from the Resources tab (visibility only in v1); gear-manager migration to the shared picker (works as-is; cleanup pass later); adding glyphs to item-icons.js.

## 1 · Architecture (approved)

- **One storage, two pickers.** The icon choice lives in the `characters` row, never forge-local. Owned rows (inventory items, custom action-editor rows) carry `icon` inline — items already do, and gear-manager already writes it. Derived rows (spells, class features — regenerated by Soul Shards on level-up) use `structural.iconOverrides`, a flat map keyed by stable tile id (`spell_healing_word`, `cf_second_wind`, `wpn-longsword` never needed — weapons are owned). Overrides survive re-derivation. Writes go through `CharacterData.save` **copy-merge**, controlling device only — the same discipline as potion consumption. Mid-fight changes paint locally at once; other devices pick them up next boot.
- **Two registries, zero name collisions.** `spell-icons.js` must not reuse any glyph name in `item-icons.js` (a smoke enforces this). The resolver checks SpellIcons first, falls through to ItemIcons, so any tile can wear any glyph from either set.
- **The keyword map demotes to a seeding layer.** `ICON_KEYWORDS` in forge-kit-derive.js remaps to real registry glyph names and only fires when no explicit choice exists. Spell defaults come from spell-icons categories; item/weapon defaults come from `ItemIcons.iconFor()` unchanged.
- **Descriptions resolve per tab, lazily where needed.** Attacks: format what the tile already carries (damage stack, to-hit, rider). Feats/universals/class features: `desc` rides the tile today. Items: `desc`/`entries` off the inventory row. Spells: the gap — full text lives in the spell data `soul-shards-data.js` reads; the drawer lazy-loads on first open and caches for the session. **Pre-build gate: read `loadSpellMeta`'s actual shape in the repo before writing the loader** (house rule: repo source before assumptions — the entries format and fetch path are assumptions until diffed).
- **Resources are a projection, not a system.** resource-derive.js already emits `{id, label, tag, max, die, recharge, tone, source, origin, custom}` with formula maxes resolved against the live sheet. The derive module stops dropping fields; the Resources tab is the sheet's tracker section as a third consumer. Spend stays on action costs.

## 2 · spell-icons.js contract

Same skeleton as item-icons.js. Categories (working set — final list at curation time): **fire & heat · frost & water · storm/lightning/force · holy & healing · mind & enchantment · illusion & shadow · necrotic & poison · wards & buffs · movement & maneuvers · sound & verbal · nature & beast · arcane & misc**. Target ~120–150 glyphs: every spell on the four PCs' current lists gets a genuinely fitting default (Vicious Mockery lands in sound & verbal), and the picker stays browsable rather than a scroll marathon. `iconFor(spellLike)` does name-keyword → school/category → arcane-misc default. Attribution line added to the repo alongside item-icons' existing one.

## 3 · The resolver (forge-kit-derive.js)

```
resolveIcon(tile, structural):
  1. tile._src.icon           (inline — items & custom actions)
  2. structural.iconOverrides[tile.id]
  3. ICON_KEYWORDS seed       (name match → registry glyph name)
  4. category default         (SpellIcons.iconFor / ItemIcons.iconFor by tile kind)
  5. null → letter tile       (never blank — Chonkalius rule)
```

Exported so forge-hud.js and any future surface share it. forge-hud.js renders `iconSvg(name, 30)` from whichever registry owns the name. Weapon attack tiles pass their `_src` inventory row straight to `ItemIcons.iconFor` — a custom icon picked in gear-manager appears on the bar with zero extra plumbing.

## 4 · Resources surface

**Top row (curated):** economy pips (◆ action / ▲ bonus) + move bar always; then up to 3 pip groups, priority: spell-slot pools by level, then class resources by `origin` (class → subclass → race → custom); overflow renders a `+N` chip that jumps to the Resources tab. Vesperian fits whole; Líadan overflows gracefully.

**⬡ Resources tab:** horizontal-scroll cards, one per pool. Card face: label · pip row (current/max) · die badge when present (Bardic d6) · origin accent (same tones as the sheet tracker). Tap expands in place: recharge text ("short or long rest"), max formula in words ("proficiency bonus per long rest"), source stamp, and spend-this-fight count derived from the local cost ledger. Cards are read-only in v1. The tab row's "tabs are sources" purity is knowingly bent — End Turn already lives there (M's call: discoverability wins).

## 5 · Detail drawer

Long-press (touch) / right-click (desktop) on any tile in any tab → card anchored above the bar. Anatomy: name + cost line (economy mark, resource cost, to-hit/DC, concentration badge) · full description body · **Change icon** (opens the picker inline) · **Hide from bar** (writes the sheet editor's own `_hidden` through the override object — the sheet and forge stay one truth). Esc / tap-outside closes. Greyed tiles open the drawer too (the drawer explains *why* they're greyed — Grapple/Shove's "contested checks are a later bite" moves from tooltip to drawer). Feats tab tiles already promised a detail drawer in the bite-2 spec (§2) — this is that drawer, generalized.

## 6 · icon-picker.js (shared widget)

Extracted from gear-manager's proven pattern: swatch showing current glyph · "Change icon" toggle · grouped grid (`CATEGORIES` from both registries; spell groups listed first when editing a spell/feature tile, item groups first for weapons/items) · tap-to-select fires a callback with the glyph name. The widget takes `{registries, current, onPick}` and owns no storage — hosts decide where the pick lands (inline `icon` vs `iconOverrides`). Hosts in this arc: the forge drawer and the sheet action editor (one new row in the existing edit form). Gear-manager untouched.

## 7 · Error handling

Registry script missing → resolver skips to letter tiles, bar narrates once ("icons unavailable — using letters"). `iconOverrides` naming a glyph that no longer exists → treated as no override (falls through the chain), never a broken image. Spell-text lazy-load failure → drawer shows the cost line + "full text unavailable" note, stays open, non-fatal. Picker save failure → narrated on the drawer, choice reverts visually. Duplicate glyph name across registries → build-time smoke failure, never a runtime surprise.
