- **A write policy granted `to authenticated` gates on "logged in," not "is a member."** With open OTP that's anyone ŌåÆ gate party/character WRITES on `public.is_member()` (overseer/dm/player), not bare `authenticated`.

Call overseer-gated RPCs from the BROWSER with the user's session token; reserve the service key for reads the client can't do (e.g. listing `auth.users`).

**Editor / rich-content layer ŌĆö decision (June 24): stay on Quill for now; TipTap parked for a future "connected content" layer.** The chronicle's `@`-mention NPC/location linking is **custom code on Quill** (colored spans tagged `data-mention-type`, found by color + tracked in a `mentionMap`; `#` tags the same way) ŌĆö NOT the `quill-mention` lib, so it's **reusable on any new Quill surface** (the at-a-glance sheet's **Notes** tab will reuse it). **TipTap (ProseMirror, MIT, headless) was evaluated** and is genuinely better *for a connected layer*: mentions are structured **nodes** (`{id,type,label}`), not color-spans ŌåÆ clickable + **queryable** refs (wiki-style backlinks ŌĆö "every entry/journal/quest that mentions this NPC/location"), clean text+image+ref content, and one reference system across every surface. **Why parked, not adopted now:** ToK is no-build vanilla; TipTap's own "vanilla" path assumes a bundler, and the genuine no-build route is CDN-via-ESM ŌĆö works but finicky (duplicate `prosemirror-model` ŌåÆ *"adding different instances of a keyed plugin"*, fixed with an **import map**); it's headless (build the toolbar yourself); and full consistency would eventually mean **migrating the chronicle off Quill**. **Revisit TipTap IF/WHEN M commits to the broader vision** ŌĆö FeedŌåÆChronicle revamp + DM/player **image submission** + in-game **journaling** + a **Quest Tracker**, all sharing one unified, queryable NPC/location reference system ŌĆö at which point the one-time CDN/import-map setup amortizes across many surfaces and the structured-reference graph is the actual payoff. **First step when revisited:** pilot TipTap on ONE reference-heavy surface (journaling), prove the import-map load + a shared mention extension, *then* extend it and migrate the chronicle. **Until then: Quill everywhere, reuse the existing mention machinery.**

**Lesson pinned ŌĆö the inject path compounds the `.js` cache gotcha.** A dynamically-injected module can keep serving a *stale* copy on an inject-only page even when the static-`<script>` pages already have the fresh one, and a **truthy-but-partial global silently passes a bare presence check**.
