# Archive

Historical records are preserved here for provenance and prior rulings. They are not the
current execution authority.

- `context/` — superseded project and subsystem handoffs.
- `plans/` — completed implementation and design plans.
- `patches/` — historical patch bundles and application notes.
- `validation/` — historical inventories, manifests, checksum lists, and validation logs.

Use the root `CONTEXT.md` and `CONTEXT_Forge.md`, then `docs/handoffs/forge/`, for current work.
